Project snapshot - reviewed 2026-05-30

Lineage is an evolving ecology sandbox with local perception, heritable bodies, neural behavior, and repeatable experiments.

The project combines a deterministic C# simulation core, a headless CLI runner, a Godot viewer, and a local web launcher. Creatures live in continuous 2D worlds, sense food and neighbors through limited local cues, make decisions through evolvable neural brains, reproduce through eggs, and leave behind lineage, species, geography, and behavior reports.

This page is a human-facing overview. It summarizes the current implementation, highlights the important option surfaces, and links back to the markdown files and source modules that should remain the source of truth.

Architecture

Lineage is split so the simulation rules can run deterministically outside the viewer. Godot is a visualization and editing surface; the CLI is the repeatable experiment runner; the core library owns state, systems, scenarios, brains, snapshots, and analysis helpers.

Simulation Tick Loop

The default neural pipeline is an ordered set of systems. A fixed delta advances the world, and Godot can run that loop at viewer speed while CLI runs it headlessly for experiments.

The SimpleForaging pipeline remains as a reference controller, but current authored scenarios use the neural path.

Entities And Energy Flow

Creatures carry hot state such as position, heading, velocity, energy, health, age, lineage IDs, gut contents, senses, action outputs, egg reserve, and brain/genome IDs. Eggs are separate entities. Resources are plant or meat patches.

EntityCurrent role
CreatureMoves, senses, eats, digests, lays eggs, attacks, mutates through descendants.
EggStores offspring genome/brain, energy, health, incubation age, and environmental exposure.
Plant resourceRegrows, depletes, enters dormancy, disperses locally, or relocates within original habitat biome.
Meat resourceCreated from death, decays, can become stale/rotten, emits meat and rot scent.

A known reporting gap is true living biomass. Current CSVs include total_creature_energy, egg energy, and resource calories, but not a first-class body-biomass time series.

World And Ecology

Worlds are bounded continuous 2D spaces. Biomes are grid-backed lookup layers with eight canonical categories: Desert, Scrubland, Grassland, Fertile, Forest, Wetland, Tundra, and Highland. Generated noise, natural-climate maps, bands, edge ladders, corridors, and reusable manual map artifacts can all drive the layout. Biomes influence food density, regrowth, movement cost, speed, vision range, basal cost, seasonal response, habitat quality, and egg environmental exposure. Obstacles can be generated or loaded from reusable maps and block or deflect movement.

Implemented World Pressure

  • Legacy names still load: Barren maps to Desert, Sparse maps to Scrubland, and Rich maps to Fertile.
  • Natural-climate maps derive biomes from moisture, forest fitness, fertility, elevation-like variation, and smoothing.
  • Resource void borders prevent plant growth near world edges.
  • Local fertility cells can be depleted by plant use and recover over time.
  • Depleted plants remember their original habitat biome before relocation.
  • Seasons can apply global or region-dependent fertility shifts.
  • Obstacle maps include barriers with gaps, scattered rocks, and manual blocked-cell layouts.

Reusable map artifacts and launcher-side map painting now exist. Future world work is planned around richer water, lakes/seas, hazards, better map libraries, richer terrain layers, more plant types, and seed dispersal.

Creatures And Genes

A creature's genome is compact but already covers body, movement, sensing, metabolism, reproduction, diet, and combat. Juveniles scale toward adult traits through the growth helper, so offspring investment and maturity age affect early survival. Effective mutation pressure is now owned by the scenario/world at reproduction time, while older genome mutation fields remain compatibility data.

Senses And Inputs

The current neural schema has 228 inputs. The old nearest-food and nearest-creature aggregate direction shortcuts were removed from the active brain contract; older 239-input brains are migrated by dropping those slots and preserving sector, contact, scent, terrain, quality, similarity, and habitat weights. Schema v3 adds grab pressure/direction and holding/contact cues.

Brain Structures

Brain construction now goes through BrainFactory. The world still stores the current neural genome type, but scenarios can choose between two neural architectures and several starter brain policies.

HybridNeural: the default architecture. Direct input-to-output weights are active, optional hidden nodes can also contribute, and current starter profiles use 4 hidden nodes. With 228 inputs and 8 dense adapter outputs, the direct layer has 1,824 weights; a 4-node hybrid starter stores 2,768 total weights.
HiddenLayerNeural: optional architecture. Direct weights remain in the genome layout for compatibility, but behavior is routed through hidden nodes. The default is 8 hidden nodes, with relay nodes created from starter outputs.

Output Contract

  • Move forward
  • Turn
  • Eat intent
  • Reproduce intent
  • Attack intent
  • Grab intent
  • Dense memory write forward
  • Dense memory write right

The first six outputs are universal physical action intents. The two memory-write slots are architecture-owned internal outputs used by the current dense neural adapter; future recurrent, sparse, or plastic brains can own memory in their own runtime state.

Starter brain kinds include SeedForager, SectorForager, ExplorerForager, OpportunisticForager, ScavengerForager, FreshnessAwareScavenger, ForagerPredator, and RandomPerFounder.

Performance And Scaling

Recent performance work moved the simulator from dense resource scans toward time-sliced sensing, array-backed spatial cells, persistent dirty resource/egg indexing, dormant plant exclusion, compact report sampling, configurable neural and sensing parallelism, optional stale-sense neural action reuse, optional extinct payload pruning, and hidden-layer short-circuiting where possible.

Current Scaling Notes

  • Default spatial cell size is 64; Long Run Performance uses 128.
  • Expensive world sensing refreshes every WorldSenseIntervalTicks, default 10, with close-range exceptions.
  • Stats snapshots default to statsSnapshotIntervalTicks 300 in checked-in scenarios.
  • Neural controller evaluation defaults to 8 worker threads; sensing defaults to 4.
  • Plants, meat, eggs, and creatures use spatial indexing for local queries.
  • Creature cells still rebuild each tick; static and semi-static resource indexing has been optimized more heavily.
  • Set neural and sensing thread counts to 1 for conservative single-threaded comparisons.
Long-run stats cadence: checked-in scenarios default to statsSnapshotIntervalTicks 300. Use lower values when diagnosing a mechanic, and use 54000 for performance-focused long runs where trend precision is less important. Coarser values reduce CSV, report, and saved-snapshot history size, but make graphs, species trends, lineage trends, and latest-snapshot diagnostics less temporally precise; they do not change simulation behavior or lineage recording.

Roadmap Themes

The next major design frontier is not only "more features"; it is adding ecological handles that create real reasons for different strategies to exist.

Brain evolution
Generic brain interface, recurrent/memory-owning brains, topology evolution, brain complexity costs, and better behavior fingerprints.
Ecology
Multiple plant types, seed dispersal, water as survival pressure, insects, hives, plant toughness/toxicity, nutrient categories, and eventually self-sustaining plant communities.
Interaction
Kin scent, egg identity, guarding, carrying, grabbing, parasites, social tolerance, aggression, and imperfect recognition.
World generation
Reusable map polish, elevation, moisture, richer water, temperature, hazards, authored maps, and editable terrain layers.
Population health
Biomass, energy flow, reproductive reserve, bottleneck reporting, overshoot crashes, and stability windows.
Run management
Launcher polish, compact long-run retention, species/brain catalogs, lineage tree, comparison packets, tags, notes, and Godot handoff.
