Viewer Controls

  • S: collapse or expand the scenario launcher
  • Space or P: pause or resume
  • R: reset the current seed
  • N: create a new seed and reset
  • F: fit the view to the world
  • G: follow the selected creature at a closer map zoom
  • B: show or hide biome colors
  • C: cycle creature color mode
  • + / -: change simulation speed
  • Arrow keys: pan the map camera
  • Shift + arrow keys: pan faster
  • Mouse wheel: zoom at cursor
  • Right or middle mouse drag: pan
  • Left click: select a creature

Scenario Launcher

The launcher panel lets you edit all shared scenario settings, then launch them visually in Godot or run a headless CLI experiment from the same values.

  • Launch/Restart: restarts the Godot viewer using the edited settings
  • Load: loads a scenario JSON file into the editor and restarts the viewer
  • Save / Save As: writes the edited scenario as CLI-compatible JSON
  • Collapse / Expand: shrinks the launcher to a small strip and moves realtime stats into the left column
  • Run CLI: saves the edited scenario to out/godot_cli_scenario.json, then runs Lineage.Cli with the requested tick count, stats CSV, HTML report path, and final snapshot path
  • Write Current Report: writes an HTML report from the simulation currently running in Godot, without rerunning the CLI
  • Open Report: opens the last successful CLI or viewer HTML report in your default browser
  • Snapshot JSON: sets where a Godot-triggered CLI run writes its final simulation snapshot
  • Checkpoint interval: writes loadable snapshot checkpoints every N ticks; use 0 to disable
  • Checkpoint folder: sets where periodic checkpoint JSON files are written
  • Load Snapshot File: loads a saved simulation snapshot into Godot for inspection
  • Load Checkpoint File: opens the checkpoint folder so you can choose a saved checkpoint
  • Load Last Snapshot: loads the final snapshot from the most recent successful Godot-triggered CLI run
  • Load Latest Checkpoint: loads the most recent checkpoint from the last successful Godot-triggered CLI run

Scenario Files

Scenario JSON files describe the world size, seed, pipeline, initial population, resource density per 1M world area, biome mode, biome cell size, resource void border, food abundance, regrowth, depleted-resource relocation, initial brain mode, per-founder random brain weights, metabolism, movement cost, eating rate, diet adaptation, death-meat output, meat decay, vision range and angle, trait upkeep costs, optional egg upkeep, egg production rate, egg incubation, maturity age, mutation strength, trait/brain mutation rates, reproduction threshold, and bite damage, bite energy cost, bite reach, and snapshot interval. The current starter scenario is:

The saved starter and pressure preset scenarios currently use a 2000 x 2000 world.

.\scenarios\balanced-foraging.json

Current pressure presets:

  • .\scenarios\gentle-foraging.json: abundant food, smaller void border, lower basal/movement pressure, and moderate reproduction delay
  • .\scenarios\balanced-foraging.json: baseline food, void border, energy pressure, longer maturity, and higher reproduction threshold
  • .\scenarios\harsh-foraging.json: sparse food, wider void border, slower regrowth, and higher basal/movement pressure

Large worlds need proportionally more resources or longer-lived creatures. The Godot viewer now treats the center view as a map camera: use the mouse wheel to zoom, arrow keys or right/middle mouse drag to pan, F to return to the full-world view, and G to follow a selected creature. The scale bar in the lower-right of the map shows the current world-unit distance. Biomes are generated from the scenario seed; richer biomes attract more initial and relocated resource patches. The resource void border is a no-plant/no-regrowth edge zone. Press B to toggle the biome and void underlay.

Creatures do not get direct biome fertility or void-border knowledge. Food direction and proximity are only exposed for edible resources inside the creature's current vision cone, while the brain receives visible food density as a local crowding cue. Selecting a creature draws that vision cone. Vision range and vision angle are heritable traits, and both have upkeep costs so maximum-range 360-degree vision is not free.

Food currently has two kinds: plants and meat. Plants regrow and relocate when fully depleted; meat is left by dead creatures, decays, and disappears. Eggs are also edible meat-like targets: they can be seen only inside the vision cone, use meat digestion efficiency, and count as egg predation deaths if fully consumed. Living creatures can also be sensed as prey inside the vision cone. A creature with attack intent can bite another creature at body-contact range; biting costs energy, deals health damage, and injury deaths become meat. A heritable diet value controls digestion: plant specialists get strong plant calories but weak meat calories, meat specialists mirror that, and omnivores sit in the middle. The balanced starter begins near herbivore behavior, with about 92% plant digestion and 28% meat digestion. Neural brains receive both a generic diet-weighted best-food cue and separate plant/meat density, direction, and proximity cues. This keeps the information local while allowing lineages to evolve different foraging responses as their digestion shifts.

The live HUD and HTML reports include foraging diagnostics: share seeing food, share touching food, share eating on the sampled tick, average visible food density, calories eaten per second, plant/meat resource calories, egg predation deaths, visible prey density, attacking creatures, attack damage per second, average time since last meal, and average effective vision range/angle.

Resource and creature rendering are zoom-aware. Close views draw individual entities. Wide or crowded map views switch to density cells so very large worlds do not try to paint every plant patch or creature body.

The HUD includes performance telemetry: TPS is actual simulation ticks processed per real second, Frame is average frame time, v is visible/estimated entities in the current view, and d is the number of individual entities or density cells drawn.

Save the resolved scenario used by a run:

dotnet run --project .\src\Lineage.Cli\Lineage.Cli.csproj -- --seed 99 --creatures 40 --resources-per-million-area 80 --save-scenario .\out\scenario_99.json --no-output

Useful Options

  • --scenario <path>: load a scenario JSON file
  • --save-scenario <path>: save the resolved scenario before running
  • --ticks <n>: number of simulation ticks to run
  • --seed <n>: override deterministic scenario seed
  • --pipeline <neural|simple>: override controller pipeline
  • --creatures <n>: override initial creature count
  • --resources-per-million-area <n>: override initial resource density; the actual patch count scales with world area
  • --resources <n>: legacy absolute resource count override; converted to density for the resolved world
  • --snapshot-interval <n>: override stats snapshot interval
  • --output <path>: stats CSV output path
  • --lineage-output <path>: lineage CSV output path
  • --traits-output <path>: final trait summary CSV output path
  • --founders-output <path>: founder summary CSV output path
  • --generations-output <path>: generation survival CSV output path
  • --lineage-trends-output <path>: founder lineage trend CSV output path
  • --report <path>: HTML run report output path
  • --save-snapshot <path>: final simulation snapshot JSON path that can be loaded back into Godot
  • --checkpoint-interval <n>: save loadable checkpoint snapshots every N ticks
  • --checkpoint-dir <dir>: directory for checkpoint JSON files; defaults near the snapshot, report, or stats output
  • --batch-report <path>: HTML comparison report output path; uses gentle/balanced/harsh presets when no batch scenarios are listed
  • --batch-scenario <path>: add a scenario to a comparison run; repeat for multiple scenarios
  • --batch-output-dir <dir>: per-run batch CSV/report output directory
  • --no-output: run without writing CSV files

Help

dotnet run --project .\src\Lineage.Cli\Lineage.Cli.csproj -- --help

The CLI currently writes a global stats CSV, a lineage CSV, a final living trait summary CSV, a founder lineage summary CSV, and a generation survival CSV, plus a lineage trend CSV. If only --output is provided, the other filenames are derived from it. The stats CSV includes plant and meat resource counts/calories plus egg predation deaths and foraging diagnostics such as food-seeing share, plant-seeing share, meat-seeing share, food-contact share, eating share, visible food/plant/meat/prey density, attacking share, attack damage per second, calories eaten per second, time since last meal, and average vision range/angle. If --report is provided, it also writes an HTML report with outcome totals, foraging diagnostics, biome resource distribution, diet/digestion summaries, trend notes, diagnostics, dominant lineage snapshots, top founder lineages, final living trait summaries, and generation survival. If --save-snapshot is provided, the final world state can be opened from the Godot launcher's Load Snapshot File button. If --checkpoint-interval is provided, each checkpoint JSON is the same loadable snapshot format and appears in the HTML report.

Current Notes

  • The viewer and CLI share the same default scenario factory.
  • The simulation already has mutation, inheritance, reproduction, death, and lineage records.
  • Reproduction lays eggs. Mature creatures first build a reproductive reserve from surplus energy, then neural brains can decide when to deposit a completed egg with the mutated offspring genome and brain. Eggs have investment-derived health, can take environmental exposure damage in poor locations, can be eaten as meat-like food by other creatures, incubate for a heritable duration, and hatch into lineage-tracked creatures if viable.
  • Juveniles grow toward adult body size, speed, and eating capacity, and cannot reproduce before maturity.
  • Food search is local and imperfect: creatures target only visible food inside their vision cone and receive visible food density rather than map fertility.
  • Dead creatures leave meat resources. Living creatures can now bite one another at close range when their brain outputs attack intent. Diet adaptation is heritable, so lineages can evolve from mostly herbivory toward omnivory or meat specialization when the scenario makes that viable.
  • Evolutionary pressure is configurable through biome-shaped food density, no-plant void borders, regrowth, death-meat output, meat decay, bite damage/cost/reach, metabolism, movement cost, shared trait upkeep, vision range/angle upkeep, optional egg upkeep, egg exposure damage, egg production rate, offspring investment, egg incubation, maturity age, reproduction threshold, and body-size upkeep.
  • Offspring investment sets egg sturdiness and hatchling birth quality; explicit fat, strength, and clutches can build on this later.
  • Next simulation work should tune combat in longer preset runs, then likely add attack/defense genes, carcass scent, or stronger r/K tradeoffs.
